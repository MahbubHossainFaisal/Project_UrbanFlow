## 🛠️ Problems Faced & Solutions

### 1. The "Lost Keys" (Directory Error)
*   **Problem**: Running `dbt` commands from the project root failed because `dbt_project.yml` was not found.
*   **Solution**: Established the standard of running dbt commands from within the `dbt/urbanflow` directory or using the `--project-dir` flag.
*   **Command**: `cd dbt/urbanflow`

### 2. Jinja Syntax Error
*   **Problem**: `generate_schema_name.sql` macro failed with a nesting error.
*   **Solution**: Identified a missing `{% endif %}` tag in the Jinja logic and corrected it.

### 3. The "Infinite Trip" (Timestamp Precision Issue)
*   **Problem**: The Silver model was loading 0 rows despite Bronze having 5.9M rows.
*   **Diagnosis**: 
    - The raw Parquet timestamps were in **microseconds**, but Snowflake interpreted them as **seconds**, pushing dates to the year 50,000+.
    - Our business rule `WHERE duration <= 180 minutes` filtered out every row because the "years" of duration failed the check.
*   **Solution**: Updated the SQL logic to extract the epoch seconds, divide by 1,000,000, and re-cast to `TIMESTAMP_NTZ`.
*   **Code Fix**: `TO_TIMESTAMP_NTZ(EXTRACT(EPOCH_SECONDS FROM TPEP_PICKUP_DATETIME) / 1000000)`

### 4. Snowflake Concatenation
*   **Problem**: `CONCAT()` was used with 3 arguments, which is non-standard in Snowflake.
*   **Solution**: Switched to the standard `||` operator for building the `trip_id` surrogate key.
